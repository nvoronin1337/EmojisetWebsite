Small Dataset

Historical Search (HS) / Real Time Filter (RTF) / Random Sample (RS)

Keywords (HS, RTF)

Users may type any combination of words or emojis they would like to search for.
If a tweet collected does not contain the specified keyword(s), it is because it is a retweet of a tweet that does contain the information.
Having emojis side-by-side will look for the specific emoji sequence.
Emojis and words can be inputted together.
Separating keywords with a comma is considered an OR statement. Having a space between keywords is considered AND.
ie: socks shoes, 🧤, pants 👗👔 = socks AND shoes OR 🧤 OR pants AND 👗👔

Discard Tweets Without Emojis (HS, RTF, RS)

Setting the switch to 😊 only pulls tweets with emojis, having it set to ❌ pulls all tweets regardless if they have emojis or not.

Tweet Amount (HS, RTF, RS)

Users can input any positive integer with a maximum value of 10000, if a user leaves the entry blank only 100 tweets will be pulled.

Date (HS)

The user may gather tweets within a specific period of time. The default is all tweets within the past week.
With a standard API key, tweets can only be pulled from up to a week back.
With a premium or enterprise API key, tweets can be pulled from any time.

Language (HS, RTF)

Select the language of the tweets you would like to gather.

Verified Users (HS)

Toggle if tweets are only from verified users or all users.

From/To/Mentioned User (HS, RTF)

From: Gather tweets from specified users.
To: Gather tweets that were sent to the specified user.
Mentioned: Gather tweets where the specified users are mentioned.

Location (HS, RTF)

Collect tweets given the radius around the specified longitude and latitude.

Likes (HS)

Gather tweets within the specifed range of likes. Default minimum is 0, default maximum is infinity.

Post Type (HS)

Most recent: searches for tweets in reverse chronological order.
Most popular: Searches for tweets from most to least popular.

Settings (and/all) (HS)

Decides whether the settings used above will apply AND or OR to one another.
If "all" is selected, all of the settings' must be fulfilled for the tweet to not be discarded (AND).
If "any" is selected, only 1 of the fields above must be fulfilled for the tweet to be eligible (OR).

Once the query is finished:

💾Query : Saves the query onto your account for use in the large dataset.
❌ : Removes the tweets from the page.
Download full results : Downloads the full list of tweets as an xls file.

Large Dataset

Saved Queries

Select a saved query from the small dataset to use for a large dataset.
Queries are saved onto a user's account, and can be deleted manually from here.
A query MUST be saved from the small dataset in order to be used for the large dataset search.

Progress / Currently Running Task

Information about the progress and result of the query is stored in the task_info.txt file.

Main Output File

The file name is extracted_data.csv.

File Name

Set a custom name for the output file.

Created At

Create a column in the output file with the date the tweet was created at.

Tweet URL

Create a column in the output file with the URL of the tweet.

Tweet ID

Create a column in the output file with the ID. The ID is at the end of the tweet's URL.
ie: twitter.com/USER/status/123456789

Username

Create a column in the output file with the name of the user who posted the tweet.

Tweet Text

Create a column in the output file with the text of the tweet (excluding links).

Emojiset

Create a column in the output file with the emojis from the tweet.

Hastags

Create a column in the output file with the hashtags associated with the tweet.

URLs

Create a column in the output file with the links from the tweet.

Image URLs

Create a column in the output file with the image links from the tweet.

Additional Files

The file name is full_tweets.json.

Full Tweet Data

Provides all possible data from the tweet.

Images

Provides the image URLs for the tweet.

Hashtags Used Together

Provides the hashtags used within the tweet.

Hashtags the User has Utilized

Provides the hashtags used in unrelated tweets by the user who posted the tweet.

Mentioned Users

Provides the users mentioned in the tweet.

Users Other Users Mentioned

Provides the users that other users mentioned in response to the tweet.

Users Who Posted or Retweeted

Provides the users who posted or retweeted the tweet.
